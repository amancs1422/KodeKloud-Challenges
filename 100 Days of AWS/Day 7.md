# Day 7 : Change EC2 Instance Type
## Step 1 : Search for EC2 in the AWS Console and access the instances dashboard.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_1.jpg)
## Step 2 : The instance is already up and running and the type is t2.micro.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_2.jpg)
## Step 3 : In running state the "Change Instance Type" option is greyed out as shown below.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_3.jpg)
## Step 4 : The instance can be stopped by using the below option.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_4.jpg)
## Step 5 : Once the instance is completely stopped the "Change Instance Type" option is enabled again.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_5.jpg)
## Step 6 : Choose the t2.nano instance type as per the requirement.
![](https://github.com/amancs1422/KodeKloud-Challenges/blob/65fc9dde40a2e672bc76de3947c81ee8c1cbbc01/100%20Days%20of%20AWS/Images/Day%207_6.jpg)
## Step 7 : After confirming the change is made properly, start the instance again.
